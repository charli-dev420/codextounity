# Contributing / Contribution

This repository is maintained by a single developer and is an experimental prototype, not broadly tested across machines. Contributions are welcome, but review can be slow and there is no guarantee of any kind, including stability, support window, roadmap delivery or installer reliability.

Ce depot est maintenu par un seul developpeur et reste un prototype experimental, globalement non teste sur la diversite des machines. Les contributions sont bienvenues, mais la revue peut etre lente et il n'y a aucune garantie, notamment aucune promesse de stabilite, de delai de support, de roadmap tenue ou de fiabilite des installeurs.

## What Helps

- Reproducible bug reports with OS, GPU, Python, Node, Docker/WSL and Unity versions.
- Installer feedback, especially failed preflight checks and unclear manual steps.
- Documentation fixes, beginner setup tips and screenshots with secrets removed.
- Small pull requests with one focused change.
- Clear notes about external licenses, model access requirements or upstream changes.

## Ce qui aide

- Rapports de bug reproductibles avec OS, GPU, versions Python, Node, Docker/WSL et Unity.
- Retours sur l'installeur, surtout les preflights qui echouent et les etapes manuelles peu claires.
- Corrections de documentation, astuces de setup debutant et captures sans secrets.
- Pull requests courtes avec un seul objectif.
- Notes claires sur les licences externes, acces modeles ou changements upstream.

## Before Opening A Pull Request

1. Do not include local paths, usernames, tokens, model files, generated meshes, screenshots with secrets or machine-specific logs.
2. Run:

```powershell
.\scripts\scan_private_leaks.ps1
.\scripts\validate_plugin.ps1
.\scripts\smoke_app.ps1 -SkipUnityBatch
```

3. If you changed installers, also run:

```powershell
.\bootstrap\install.ps1 --help
.\bootstrap\install.ps1 --dry-run --target windows --profile auto
.\bootstrap\install.ps1 --validate-only --target windows --profile auto
docker compose -f docker\compose.yaml config
```

4. Explain what was tested and what was not tested.
5. Remove `proof/*.json` files generated by validation before opening the PR.

## Avant d'ouvrir une pull request

1. Ne pas inclure de chemins locaux, noms utilisateur, tokens, fichiers de modele, meshes generes, captures avec secrets ou logs propres a votre machine.
2. Lancer :

```powershell
.\scripts\scan_private_leaks.ps1
.\scripts\validate_plugin.ps1
.\scripts\smoke_app.ps1 -SkipUnityBatch
```

3. Si vous avez modifie les installeurs, lancer aussi :

```powershell
.\bootstrap\install.ps1 --help
.\bootstrap\install.ps1 --dry-run --target windows --profile auto
.\bootstrap\install.ps1 --validate-only --target windows --profile auto
docker compose -f docker\compose.yaml config
```

4. Expliquer ce qui a ete teste et ce qui ne l'a pas ete.
5. Supprimer les fichiers `proof/*.json` generes par les validations avant d'ouvrir la PR.

## Contribution Rights

Only submit work that you have the right to contribute. Do not copy code, model files, workflows or assets from third-party projects unless their license clearly allows it and the source is credited.

Because this project is currently unlicensed, a formal project license may be added later before broader reuse or distribution. Opening a PR does not transfer ownership of your unrelated work, but you agree that the maintainer may include your submitted changes in this repository if accepted.

## Droits de contribution

Soumettez uniquement du contenu que vous avez le droit de contribuer. Ne copiez pas de code, fichiers de modele, workflows ou assets de projets tiers sauf si leur licence l'autorise clairement et que la source est creditee.

Comme ce projet est actuellement sans licence open source, une licence formelle pourra etre ajoutee plus tard avant une reutilisation ou distribution plus large. Ouvrir une PR ne transfere pas la propriete de vos travaux sans rapport, mais vous acceptez que le mainteneur puisse inclure vos changements soumis dans ce depot s'ils sont acceptes.

## Review Expectations

This is a solo-maintained experimental prototype. A PR may be delayed, closed, split, rewritten, or rejected if it increases maintenance cost, adds unclear external dependencies, weakens safety checks, or does not match the current direction.

## Attentes de revue

Ce projet est un prototype experimental maintenu par une seule personne. Une PR peut etre retardee, fermee, decoupee, reecrite ou refusee si elle augmente trop la maintenance, ajoute des dependances externes peu claires, affaiblit les controles de securite ou ne correspond pas a la direction actuelle.
